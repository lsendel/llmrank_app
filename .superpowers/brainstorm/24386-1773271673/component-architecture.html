Hooks
State Management
use-onboarding-wizard.ts
Extended with new discovery state + actions. Existing Profile/Website logic untouched.
+ SET_BUSINESS_GOAL
+ TOGGLE_PERSONA
+ ADD_KEYWORD / REMOVE_KEYWORD
+ SET_COMPETITORS / TOGGLE_COMPETITOR
+ ADD_MANUAL_COMPETITOR
+ DISCOVER_START / SUCCESS / FAIL
+ SET_DISCOVERY_CARD_OPEN
use-competitor-discovery.ts
NEW hook — handles AI competitor suggestion lifecycle.
• Watches crawl status from parent
• On crawl complete → calls POST /api/wizard/suggest-competitors
• Passes domain + keywords + goal for context
• Returns { suggestions, loading, error }
• Auto-triggers when competitor card opens
use-discovery-completion.ts
NEW hook — tracks which cards are completed, drives pills + footer.
• Goals: complete when selection made
• Personas: complete when ≥1 selected
• Keywords: complete when ≥1 entered
• Competitors: complete when ≥1 selected
• Returns { completedCount, cardStatuses }
use-discovery-save.ts
NEW hook — handles "Go to Dashboard" save-all logic.
• Saves goal to project (PATCH /api/projects/:id)
• Creates personas (POST /api/personas)
• Saves keywords (POST /api/keywords)
• Adds competitors (POST /api/competitors)
• Uses Promise.allSettled — partial failure OK
• Redirects to /dashboard on complete
